## What This Skill Does

Identifies connections, synergies, conflicts, and opportunities across V.tal, Nio, and Tecto that wouldn't be visible from within any single entity. This is the Chief of Staff's unique vantage point — the holding-level view.

## When to Use

- Any time a topic touches more than one entity
- When an initiative in one entity could benefit or disrupt another
- When resource allocation decisions involve trade-offs between entities
- When the user needs to advocate for a cross-entity initiative
- Proactively — I should be running this lens constantly, not waiting to be asked

## The Synthesis Framework

### The Holding Physics

```
V.tal (Infrastructure)
  │
  ├── Fiber Network ──→ enables ──→ Nio (Broadband/Retail)
  │                                    │
  ├── Subsea Cables ──→ enables ──→ Tecto (Data Centers)
  │                                    │
  └── Field Services ──→ serves ──→ Both Nio & Tecto
                                       │
                              Tecto ←──→ Nio
                          (compute for    (retail edge
                           AI/cloud)       use cases?)
```

**Key insight:** These entities share physical infrastructure but have separate P&Ls, teams, and incentives. Misalignment between shared infrastructure and separate incentives is where problems (and opportunities) hide.

### Synthesis Patterns

| Pattern | Description | Example |
|---------|-------------|---------|
| **Shared Pain** | Same problem manifesting in multiple entities | Customer service issues in both V.tal Field Services and Nio CX |
| **Upstream/Downstream** | Decision in one entity creates ripple in another | V.tal network changes affecting Nio service quality |
| **Resource Competition** | Entities competing for same scarce resource | Field technicians serving both V.tal wholesale and Nio retail |
| **Capability Transfer** | Skill or system in one entity useful to another | Nio's CX platform could serve Tecto's enterprise clients |
| **Bundle Opportunity** | Combined offering more valuable than parts | Fiber + compute + connectivity as integrated solution for hyperscalers |
| **Cannibalization Risk** | One entity's growth threatens another | If Tecto serves ISPs directly, does that affect V.tal's neutral positioning? |

## Key Questions I Should Always Be Asking

1. **Infrastructure → Retail:** Is V.tal's network roadmap aligned with Nio's growth plan?
2. **Infrastructure → Compute:** Is Tecto's data center placement optimal for V.tal's fiber and subsea routes?
3. **Retail → Infrastructure:** Are Nio's customer pain points feeding back into V.tal's service quality?
4. **Retail → Compute:** Could Nio's customer base be a channel for Tecto's edge services?
5. **Compute → Infrastructure:** Is Tecto's demand forecast driving V.tal's capacity planning?
6. **Field Services:** Is the field ops team optimized for serving both entities, or is there a priority conflict?

## Pitfalls to Avoid
- Forcing synergies that don't exist (not everything needs to be "integrated")
- Ignoring entity autonomy — Nio has independent governance for a reason
- Being naive about politics — cross-entity initiatives need sponsors in each entity
- Optimizing for holding-level value at the expense of entity-level execution
- Treating neutrality as a given — V.tal's neutral network positioning requires careful management

## Advancement Criteria
- **→ L1:** First cross-entity analysis delivered
- **→ L2:** Understanding of actual interdependencies (not just theoretical ones)
- **→ L3:** Proactively surfacing cross-entity risks and opportunities before they're asked about
- **→ L4:** Serving as the connective tissue between entities; user relies on this skill for all multi-entity decisions
