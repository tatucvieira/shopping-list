# Chief of Staff — AI Operating System

> A self-improving AI Chief of Staff for the Product & Innovation Director across V.tal, Nio, and Tecto.

---

## Architecture

Inspired by [OpenClaw's](https://github.com/openclaw/openclaw) agent identity framework, adapted for a Chief of Staff operating within Claude Code.

```
.
├── SOUL.md                 # Philosophy, values, behavioral core
├── IDENTITY.md             # Voice, presentation, operating style
├── USER.md                 # Context about the user (continuously updated)
├── MEMORY.md               # Long-term curated knowledge
├── EVOLUTION.md            # Self-improvement protocol & learning backlog
├── SESSION_PROTOCOL.md     # How each session starts, runs, and ends
│
├── memory/                 # Daily session logs
│   └── YYYY-MM-DD.md      # Append-only daily journals
│
├── skills/                 # Capabilities with maturity tracking
│   ├── strategic-analysis/
│   ├── stakeholder-briefing/
│   ├── market-research/
│   ├── decision-framing/
│   ├── communication-drafting/
│   └── cross-entity-synthesis/
│
└── context/                # Holding intelligence
    ├── HOLDING_OVERVIEW.md # Strategic briefing (full holding)
    ├── VTAL.md             # V.tal entity intelligence
    ├── NIO.md              # Nio entity intelligence
    ├── TECTO.md            # Tecto entity intelligence
    └── MARKET.md           # Market & competitive intelligence
```

## How It Works

1. **Each session**: Read core files → orient → engage → capture → curate → evolve → commit
2. **Knowledge compounds**: Every interaction adds to memory, sharpens skills, deepens context
3. **Skills mature**: L0 (Seed) → L1 (Developing) → L2 (Competent) → L3 (Advanced) → L4 (Mastered)
4. **Everything is plain text**: Human-readable, git-versioned, fully controllable

## Key Principles

- **Provoke, don't just produce** — Challenge assumptions before jumping to solutions
- **Cross-pollinate relentlessly** — Every topic gets the holding-level lens
- **Earn trust through precision** — Distill, don't dump
- **Grow into the gaps** — Track weaknesses and deliberately improve
